# arunveligatla.com

Personal portfolio for Arun Veligatla. Next.js 15 (App Router) + TypeScript strict + Tailwind v4 + MDX content. Deployed on Vercel.

- Live: <https://arunveligatla.com>
- Source: <https://github.com/arunveligatla99/portfolio>

## Quick start

```bash
npm install
npm run dev          # http://localhost:3000
```

The site runs without any environment variables in dev. The contact form
returns `{ ok: true, stub: true }` until `RESEND_API_KEY` is configured.

## Scripts

| Script              | Purpose                                   |
| ------------------- | ----------------------------------------- |
| `npm run dev`       | Next dev server, with HMR.                |
| `npm run build`     | Production build, generates static pages. |
| `npm run start`     | Serves the production build locally.      |
| `npm run typecheck` | `tsc --noEmit` (strict).                  |
| `npm run lint`      | ESLint flat config + jsx-a11y.            |
| `npm run test`      | Vitest unit/component suite.              |
| `npm run test:e2e`  | Playwright smoke + axe scan + screenshots.|
| `npm run format`    | Prettier with Tailwind class sort.        |

## Environment variables

All optional. Defaults are fine for local dev. `.env.example` is the
canonical list.

| Var                    | Default                       | Purpose                                                |
| ---------------------- | ----------------------------- | ------------------------------------------------------ |
| `NEXT_PUBLIC_SITE_URL` | `https://arunveligatla.com`   | Canonical URL used by sitemap, OG, JSON-LD, robots.    |
| `RESEND_API_KEY`       | unset                         | When set, `/api/contact` actually sends email via Resend. When unset, the form returns `{ ok: true, stub: true }` and logs the payload server-side. |
| `RESEND_FROM`          | `contact@arunveligatla.com`   | From address.                                          |
| `CONTACT_TO_EMAIL`     | `arun.veligatla@gmail.com`    | Inbox for contact-form submissions.                    |

## Deploy

Vercel detects Next.js automatically. `vercel.json` declares the redirect
from `/cv` to `/resume` and a small set of security headers. To deploy:

1. Push to `main`.
2. Vercel builds and deploys. Sitemap, robots, OG image, and per-page metadata are all generated by Next.
3. Set the env vars above in the Vercel project settings if you want real email delivery.

## Content authoring

### Add a new project case study

Create `content/projects/<slug>.mdx`:

```mdx
---
title: 'Project Title'
tagline: 'One-line tagline, max 160 chars.'
role: 'Your role'
period: '2025 to present'
domain: ['ai-ml', 'fintech']         # one or more from the enum
stack:
  - 'Python'
  - 'FastAPI'
highlights:
  - 'Quantified bullet 1.'
  - 'Quantified bullet 2.'
metrics:
  - { label: 'Latency', value: '< 800ms' }
links:
  - { label: 'GitHub', href: 'https://github.com/...' }
featured: true                       # surfaces on the home page
order: 6                             # lower = earlier in the grid
hero:
  eyebrow: 'AI / RAG'
  problem: 'One sentence.'
  outcome: 'One sentence.'
---

## Problem
...
## Approach
...
## Stack
...
## Outcomes
...
## Lessons
...
```

Frontmatter is validated by `lib/projects-shared.ts` (zod). Bad frontmatter fails the build with a precise error.

Domains enum: `healthcare`, `fintech`, `ai-ml`, `enterprise-saas`, `platform`. Add new values to `ProjectDomain` and `domainLabels` in `lib/projects-shared.ts`.

#### Author markers inside MDX bodies

Use markdown reference-link comments, not JSX comments, for author-only
notes inside an MDX body:

```mdx
[//]: # (TODO ARUN COPY: pull a representative trace screenshot if cleared.)
```

`next-mdx-remote` v6 defaults `blockJS` and `blockDangerousJS` to `true`,
so JSX expressions like `{/* ... */}` in an MDX body fail compilation.
The reference-link form renders to nothing in CommonMark / MDX, is still
greppable (`grep -rn "TODO ARUN COPY" content/`), and survives the
v6-hardened defaults.

### Add a writing post

Create `content/writing/<slug>.mdx`:

```mdx
---
title: 'Post title'
description: 'One-line description.'
date: '2026-05-07'
tags: ['rag', 'evals']
draft: false
---

Body in MDX.
```

Posts with `draft: true` are excluded from the public listing.

### Content rules (enforced by tests)

`tests/unit/content-guard.test.ts` fails the build if it finds:

- The em dash character (`—`). Use a comma, period, or rewrite.
- Banned phrases: `passionate about`, `leveraging cutting-edge`, `in today's fast-paced world`, `I'm thrilled to`.
- The words `RIF`, `layoff`, `laid off` anywhere in source or content.

```
├── site/               # SiteHeader, SiteFooter, ThemeToggle, JsonLd
├── home/               # Hero, FeaturedProjects, SkillsSummary, ContactCta
├── projects/           # ProjectCard, ProjectFilter, ProjectsGrid, CaseStudyHeader, MetricsRow, StackList

├── projects/*.mdx
└── writing/*.mdx

lib/
├── mdx.ts              # generic MDX directory loader
├── projects.ts         # server: loads + validates project MDX
├── projects-shared.ts  # client-safe: types, schema, labels
├── writing.ts          # server: loads + validates writing MDX
├── seo.ts              # metadata helpers, JSON-LD builders
└── utils.ts            # cn() helper

tests/
├── unit/               # cross-cutting (content guard)
├── setup/              # vitest setup
└── e2e/                # Playwright smoke + axe + screenshots
```

### Key decisions

- **Dark default with light toggle.** Theme persists in `localStorage`. Inline init script in `<head>` prevents FOUC.
- **No client JS for static content.** Only client islands: `ThemeToggle`, `ProjectFilter`, `ProjectsGrid`, `ContactForm`, `error.tsx`.
- **MDX with build-time shiki.** Code blocks ship as plain HTML, no client highlighting.
- **Server / client module split.** `lib/projects.ts` reads `node:fs`; `lib/projects-shared.ts` ships types and constants safely to the client.
- **Native sitemap and robots** via `app/sitemap.ts` / `app/robots.ts`.

## Lighthouse

Lighthouse smoke is a manual step. From a built site running locally:

```bash
npm run build
npm run start &
npx @lhci/cli@0.13.x autorun --collect.url=http://localhost:3000 --collect.url=http://localhost:3000/projects/policymind --upload.target=temporary-public-storage
```

Targets: ≥ 95 on Performance, Accessibility, Best Practices, SEO. LCP <
2.0s, CLS < 0.1, INP < 200ms. Latest Lighthouse outputs live in
`specs/07-verification-report.md`.

## Testing

- **Vitest** for component / unit / API route tests.
- **Playwright** for end-to-end smoke + axe-core a11y scan + viewport screenshots (375 / 768 / 1440).
- **Content guard** test enforces tone rules.

```bash
npm run test                 # vitest
npm run test:e2e             # playwright (boots npm start)
```
