## Problem

Traditional ERP platforms either force a rigid schema on every tenant or hide a slow EAV table behind a custom fields UI. Neither is acceptable when the same platform has to serve a fifty-person clinic and a thousand-person enterprise on shared infrastructure.

The constraints I gave myself:

- Single shared cluster, no per-tenant database sprawl.
- Custom columns must be queryable with the same indices and predicate pushdown as core columns.
- Tenant onboarding must reach a usable, configured environment without a human integrator.

## Approach

Three load-bearing pieces, designed as independent subsystems that compose:

1. **Tenant routing.** A connection middleware resolves tenant context per request, swaps the active schema, and short-circuits cross-tenant reads at the SQL layer rather than relying only on app-level filters.
2. **Custom Columns engine.** Tenant-defined columns serialize into typed JSON shadow tables with computed columns and covering indices. Queries compile through a type-checked DSL that produces parameterized SQL, so a tenant adding a `priority_score` column gets the same execution plan as if it shipped in the base schema.
3. **Agentic Onboarding Workflow.** A LangGraph state machine reads a questionnaire, plans schema extensions, generates seed data, and writes the per-tenant feature flag set. Each node is a checkpoint with rollback. The agent does not freelance; it picks from a registry of vetted operations.

## Stack

- **Backend:** C# / .NET 10, EF Core, Hangfire for background jobs.
- **Frontend:** Angular 17 with signals, NgRx-free.
- **Data:** MySQL 8 with JSON columns and generated indexes.
- **Cloud:** Azure App Service, Azure Container Registry, Key Vault.
- **AI / orchestration:** LangGraph for the onboarding state machine, OpenAI on Azure for natural language to schema mapping.
- **Observability:** OpenTelemetry, Serilog with structured fields, an internal trace correlator.

## Outcomes

The platform supports the design target of 1k+ tenants on a shared cluster with row-level isolation, custom columns indexed at the same tier as first-class fields, and onboarding times measured in minutes rather than days. Every CI build runs contract checks against the OpenAPI spec to catch breaking changes before deploy.

## Lessons

- A typed query builder over JSON columns is much faster to build than a full ORM extension and pays for itself the first time you have to debug a slow tenant query.
- LangGraph is most useful when the agent is constrained to a small registry of operations. The hard work is the registry, not the model.
- Per-tenant feature flags removed entire categories of release risk. A canary tenant pattern caught two regressions before a single paying tenant saw them.
