## Problem

A binary default classifier is the wrong primitive for a real lending operation. Operators ask three different questions: "when", "is the model still right", and "did our action actually help". A single model cannot answer all three honestly.

## Approach

Three modules, three contracts, one platform.

1. **Survival.** XGBoost with the AFT loss handles right-censored loans natively. The output is a survival curve per loan, not a single score, so downstream tooling can pick its own threshold by horizon.
2. **Drift.** Population Stability Index per feature on rolling windows, with feature-class-specific thresholds. Sustained drift pages, single-window noise does not.
3. **Causal.** A T-Learner over a causal forest baseline estimates per-borrower uplift from each available action. Validation is done against held-out randomized experiments, not in-sample tests.

## Stack

- **Modelling:** Python, XGBoost (AFT), CausalML, scikit-learn.
- **Pipelines:** Databricks for batch, MLflow for tracking and registry.
- **Serving:** FastAPI behind a load balancer, with feature contracts validated on every request.
- **Monitoring:** PSI dashboards, alerting on sustained drift.

## Outcomes

- Operators get a horizon-aware view of default risk instead of a binary score.
- Drift alerts catch real distributional shifts without firing on noise.
- Causal estimates are validated against actual experiments, not in-sample.

## Lessons

- Survival models are often the right answer when "when" matters more than "if".
- A drift system that fires too often gets ignored. The operator-facing default has to be precision over recall.
- Causal effects measured in-sample are not effects, they are wishes.
